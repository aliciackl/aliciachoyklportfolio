ADOOH Dashboards

Data-Driven Insights for System Optimization

Data Cleaning

Upon receiving the dataset in .SQL format, we imported it into MySQL using MySQL Workbench. To host the database locally, we utilized WAMPServer. MySQL Workbench enabled us to visualize the data schema, after which we performed comprehensive data cleaning. We then created targeted views to isolate datasets most relevant for generating actionable insights.

Final Dashboards

After the data was cleaned, we categorized it according to key metrics: storage, downloads, file access, and system issues. This segmentation enabled more focused and effective analysis for each domain.

To facilitate data exploration and initial visualization, we used Microsoft Excel to generate preliminary charts and graphs. These visuals helped summarize large datasets and identify trends. We then engaged with the client to refine requirements and incorporated their feedback into the final dashboard design.

Main Page

Full Overview

The overview dashboard is divided into four key subsections: Storage, Downloads, File Access, and Issues. Each section delivers targeted analysis of ADOOH system performance, with the goals of identifying failure points, validating client hypotheses, and understanding user behavior. Insights are presented through six pages of interactive, static dashboards.

Future overview

Leveraging machine learning techniques such as XGBoost and exponential smoothing, we forecasted trends in file access, storage activity, and projected growth. The results were distilled into percentage-based insights for clear, digestible reporting.

Sub tabs

STORAGE

We conducted account-level analysis of storage usage by querying storage log data with SQL. Time-series visualizations illustrate daily activity and cumulative storage per user. These interactive charts allow stakeholders to filter by account, identify activation dates, and monitor storage growth trajectories.

Downloads

Download log data was analyzed to identify the underlying causes of failed attempts. Stacked bar charts display success and failure rates over time, allowing us to pinpoint periods of increased failures and evaluate their impact. We also performed comparative analysis across file types to highlight performance discrepancies.

File access

We analyzed file access attempts on a per-account basis over the full data period. Error events were isolated and plotted using time-series visualizations, revealing four significant spikes. The absence of ongoing errors suggests that these were isolated, time-bound incidents rather than systemic issues.

Issues

Our investigation of storage and file system logs uncovered two major instances of user data collection anomalies, highlighting potential areas for process improvement. collection errors.
